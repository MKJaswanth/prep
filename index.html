04 / Exam Strategy

Win on Exam Day

⚡ Section-wise Strategy
V
Verbal: Read questions first, then skim RC passage. Do grammar Qs first (fastest).
R
Reasoning: Draw grids for arrangements. Skip hard puzzles, return if time allows.
N
Numerical: Use the on-screen calculator always. Estimate for DI — no exact math needed.
P
Prog Logic: Trace code line by line on scratch paper. Eliminate wrong options first.
C
Coding: Brute-force first — partial marks count. Test with examples before submit.
🧠 Smart Shortcuts
1
Memorise squares 1–25, cubes 1–15. Saves 20–30 sec per Quant question.
2
Fraction-% table: 1/3=33.3%, 1/6=16.7%, 1/7=14.3%, 1/8=12.5%, 1/9=11.1%
3
Blood relations: Always draw a family tree. Never solve in your head.
4
Arrangements: Make a grid immediately — rows = people, cols = attributes.
5
Syllogisms: Draw tiny Venn circles for each statement before evaluating.
🚫 What NOT to Do
✕
Leave questions blank. No negative marking — always attempt. Guess after elimination.
✕
Spend 3+ min on one Aptitude Q. Mark your best guess and move on immediately.
✕
Start new topics on Day 10. Only revise what you know — panic, not progress.
✕
Pull an all-nighter. Cognitive performance drops 20–30% without 7–8 hrs sleep.
✕
Skip the Day 7 mock test. Time management is a skill — it must be practiced.
🎯 Languages Allowed
Py
Python — Recommended. Fastest to write, least syntax. Best for most students.
C
C/C++ — Good for MCQ section. Use what you're most comfortable with.
J
Java — Acceptable. Slightly more verbose but valid choice.
💡 Pro tip: Use Python dict & list comprehensions. They solve most TCS problems in 3–5 lines.
03 / Quick Reference

Formula Cheat Sheet

Time & Work
A in x days, B in y days → Together = (x×y)/(x+y) days
If A is n times efficient as B → A does n parts, B does 1 part
Time Speed Distance
Speed = Distance / Time
Relative speed (same dir) = |S1 - S2|
Relative speed (opposite) = S1 + S2
Train crossing pole: T = Length / Speed
Profit & Loss
Profit% = (Profit / CP) × 100
SP = CP × (1 + P%/100)
Successive discount a%, b% = a + b - (ab/100)%
Compound Interest
A = P(1 + r/100)^n
CI = A - P
SI = (P × R × T) / 100
Difference CI - SI (2yr) = P(r/100)²
Probability
P(E) = Favourable / Total outcomes
P(A or B) = P(A) + P(B) - P(A∩B)
P(A and B) = P(A) × P(B) [independent]
P(not A) = 1 - P(A)
Percentages Tricks
x% of y = y% of x (flip trick!)
Successive %: a+b+(ab/100)
% change = (New-Old)/Old × 100
Fraction shortcuts: 1/8=12.5%, 1/6=16.7%
Permutations & Combinations
nPr = n! / (n-r)!
nCr = n! / [r! × (n-r)!]
Circular arrangements = (n-1)!
Identical items: n! / (p! × q!)
Mixtures (Alligation)
Ratio of mixing = (Dearer - Mean) : (Mean - Cheaper)
Mean price = Total cost / Total quantity
Replacement: Final = P × (1 - d/V)^n
